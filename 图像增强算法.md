


> 图像增强算法

## 自适应 HDP（Retinex）图像增强算法
该算法是在 **Retinex（SSR）** 的基础上进行工程优化，主要解决以下问题：
-   图像存在光照不均，希望增强亮暗对比，同时保持整体亮度
-   黑色区域存在大量噪声
-   黑色像素参与 `log()` 运算后容易放大噪声
---
## 算法流程

1.  输入 RGB 图像
2.  转换为 CV_32FC3 浮点图像
3.  统计全图三个通道最小的150个像素值
4.  使用 101×101 均值滤波估计背景光照
5.  计算 Retinex
    
    ```
    R = log(I+1) - log(B+1)
    ```
    
6.  忽略极暗像素参与 Retinex 运算
7.  统计 Retinex 最大值和最小值
8.  线性归一化到 0~255
9.  输出 8bit RGB 图像
---

## Step1：转换为浮点图像

```cpp
src.convertTo(imgFloat, CV_32FC3);
```

### 作用

Retinex 需要进行 log() 运算，因此必须使用浮点数据。

如果仍使用 uchar：

-   精度不足
-   对数运算误差较大
-   后续归一化困难

因此统一转换为 CV_32FC3。
## Step2：寻找最暗像素

```cpp
const int KEEP_COUNT = 150;
```

这里统计的是整个图像三个通道中最小的150个数值，而不是150个像素。

例如：

```cpp
RGB20 18 2135 40 33...
```

展开以后实际上变成：

```cpp
201821354033...
```

然后利用

```cpp
std::nth_element()
```

寻找第150小的值。最终得到

```cpp
minThreshold
```
后续所有
```cpp
value <= minThreshold
```
均认为属于极暗噪声。

### 为什么不用 sort()?提高运算时间

sort() 时间复杂度为

```cpp
O(n log n)
```

这里只需要第150小元素，因此使用

```
std::nth_element()
```

平均复杂度约为

```cpp
O(n)
```

速度更快。

----------

## Step3：估计背景光照

```cpp
blur(imgFloat, background, Size(101,101));
```

这里的大尺寸均值滤波并不是为了降噪，而是估计背景光照（Illumination）。

101×101 核能够滤除局部纹理，仅保留整体亮度变化。

得到的 background 即 Retinex 中的背景光照图。

----------

## Step4：Retinex计算

公式：

```cpp
R = log(I+1) - log(B+1)
```

对应代码：

```cpp
value = logf(src + 1.0f) - logf(background + 1.0f);
```

其中：

-   I 为原图
-   B 为背景光照

加 1 的目的是避免出现 log(0)。

----------

## Step5：忽略最暗像素

```cpp
if (pSrc[x][c] <= minThreshold)    continue;
```

这是本算法增加的工程优化。

Retinex 会放大暗区域噪声，因此直接忽略最暗150个数值，不参与计算。

这样可以降低：

-   热噪声
-   CMOS暗噪声
-   CCD黑电平噪声

最终提高增强效果。

----------

## Step6：统计输出范围

Retinex 输出通常不是 0~255，而是浮点值，例如：

```cpp
-2.31~1.75
```

因此需要统计：

```cpp
minValmaxVal
```

用于后续归一化。

----------

## Step7：线性归一化

计算比例：

```cpp
scale = 255 / (maxVal - minVal)
```

代码：

```cpp
float scale = 255.0f / std::max(maxVal - minVal, 1e-6f);
```

其中：

```cpp
1e-6
```

用于避免除零。

----------

## Step8：转换为8位图像

归一化公式：

```
value = (value - minVal) * scale;
```

最后使用

```
cv::saturate_cast<uchar>()
```

限制输出范围为

```
0~255
```

防止数据溢出。

----------


## 工程优化总结
| 优化项 | 实现方法 | 优点 | 作用 |
|--------|----------|------|------|
| 浮点计算 | `CV_32FC3` | 提高计算精度 | 避免 `log()` 运算产生较大误差 |
| 最暗像素统计 | `std::nth_element()` | 平均 O(n) | 快速获得暗像素阈值 |
| 暗像素过滤 | 跳过最暗150个数值 | 抑制暗区噪声 | 防止 Retinex 放大黑色噪声 |
| 背景估计 | `blur(Size(101,101))` | 平滑背景 | 提取低频光照分量 |
| 全局归一化 | min-max 拉伸 | 提高对比度 | 充分利用 0~255 灰度范围 |
| 数据转换 | `saturate_cast<uchar>()` | 防止溢出 | 保证输出数据合法 |



----------

## 适用场景

-   光照不均补偿
-   工业相机图像增强
-   条纹缺陷检测
-   黑色划痕检测
-   金属表面检测
-   后续边缘检测、模板匹配等视觉算法预处理
